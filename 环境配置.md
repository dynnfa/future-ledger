# FutureLedger v0 开发环境配置

本文依据 `docs/plans/v0-dividend-report-implementation.md` 整理，目标是让一个“会编程，但刚接触 Python 项目管理”的开发者可以在本机开始实现 FutureLedger v0。

v0 是一个本地 Python CLI 项目，不是 Web 应用，不需要数据库，不需要 Docker，也不需要先申请任何金融数据 API Key。主要依赖来自 Python 包，统一交给 `uv` 管理。

## 需要先安装的软件

### 1. Git

用途：

- 拉取、提交代码
- 后续配合 GitHub Actions 做 CI

验证：

```bash
git --version
```

macOS 如果还没安装开发者命令行工具，运行上面的命令时系统通常会提示安装。也可以手动执行：

```bash
xcode-select --install
```

### 2. uv

`uv` 是本项目的 Python 项目管理器，负责：

- 安装和固定 Python 3.11
- 创建虚拟环境 `.venv`
- 安装项目依赖
- 生成并使用 `uv.lock`
- 运行命令，例如 `uv run pytest`

推荐优先安装 `uv`，而不是先手动安装一堆 Python 包。

macOS / Linux：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Windows PowerShell：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

如果你在 macOS 上习惯 Homebrew，也可以：

```bash
brew install uv
```

验证：

```bash
uv --version
```

如果提示 `uv: command not found`，通常是安装目录还没有进入 `PATH`。重开一个终端，或按安装脚本末尾提示把 `~/.local/bin` 加到 `PATH`。

### 3. Python 3.11

计划指定运行时为 Python 3.11。建议让 `uv` 管理项目 Python，不要使用系统自带 Python，也不要把项目依赖装进全局 Python。

安装 Python 3.11：

```bash
uv python install 3.11
```

验证：

```bash
uv run python --version
```

期望看到类似：

```text
Python 3.11.x
```

### 4. 代码编辑器

任选一个即可：

- VS Code
- Cursor
- PyCharm
- Neovim / Vim

建议编辑器启用 Python、Ruff、Pytest 支持，但这不是运行项目的硬性要求。

### 5. Excel 查看工具

项目会生成 `.xlsx` 报告。开发和测试主要靠 `openpyxl` 自动验证，但人工检查时需要一个表格软件：

- Microsoft Excel
- WPS Office
- LibreOffice
- Apple Numbers

这不是代码运行必需项，只是方便查看输出文件。

## 不需要提前安装的软件

以下内容不需要手动全局安装：

- `akshare`
- `pandas`
- `openpyxl`
- `typer`
- `tenacity`
- `pytest`
- `ruff`

这些都应该写入 `pyproject.toml`，由 `uv` 安装到项目虚拟环境里。

也不需要：

- Docker
- Node.js / npm
- 数据库
- Redis
- Jupyter Notebook
- Web 浏览器自动化工具
- 金融数据 API Key

## 项目初始化时需要安装的 Python 依赖

实施计划中的初始化命令是：

```bash
uv init --package
uv add akshare pandas openpyxl typer tenacity
uv add --dev pytest ruff
uv lock
```

这些包的用途如下：

| 包 | 作用 |
|---|---|
| `akshare` | 获取 A 股股票列表、分红明细、历史价格 |
| `pandas` | 表格数据处理，AKShare 返回的主要就是 DataFrame |
| `openpyxl` | 写入和测试 `.xlsx` 工作簿 |
| `typer` | 构建命令行程序 `future-ledger` |
| `tenacity` | 给 AKShare 网络请求增加重试 |
| `pytest` | 写和运行测试 |
| `ruff` | 代码检查和格式化 |

如果项目已经有 `pyproject.toml` 和 `uv.lock`，新开发者通常不再执行 `uv add`，而是执行：

```bash
uv sync --locked
```

## 推荐的一次性配置流程

在仓库根目录执行：

```bash
cd /Users/dynnfa/TechPrivate/FutureLedger
uv python install 3.11
```

如果还没有 Python 项目骨架，按实施计划执行：

```bash
uv init --package
uv add akshare pandas openpyxl typer tenacity
uv add --dev pytest ruff
uv lock
```

如果项目骨架已经存在，执行：

```bash
uv sync --locked
```

然后验证开发命令：

```bash
uv run python --version
uv run ruff check .
uv run pytest
```

实现 CLI 入口后，还需要验证：

```bash
uv run future-ledger --help
uv run future-ledger dividends scan --limit 5 --output reports/dividend_rank.xlsx
```

## 网络和数据源要求

v0 使用 AKShare 抓取公开数据，主要数据来自东方财富相关接口。运行真实扫描时需要：

- 可以访问 AKShare 使用的上游站点
- 网络相对稳定
- 接受上游接口偶发超时、字段变化或空数据

开发测试不应该依赖真实网络。计划中要求默认测试使用 fixtures 和 mock：

```bash
uv run pytest
```

真实 AKShare 连通性测试应当作为手动 smoke test，或后续加到 `pytest -m live`，不要放进默认 CI。

## Python 新手需要理解的几个概念

### 不要用全局 pip 装依赖

不要这样做：

```bash
pip install pandas akshare
```

本项目应该通过 `uv add` 或 `uv sync` 管理依赖。这样每个人的依赖版本可以被 `uv.lock` 固定，CI 和本机结果更一致。

### `uv run` 等于“在项目环境里运行”

例如：

```bash
uv run pytest
```

含义是：用当前项目的 Python、当前项目的依赖来运行 `pytest`。

这类似其他生态里的：

- Node.js: `npm run ...`
- Rust: `cargo run ...`
- Ruby: `bundle exec ...`

### `.venv` 是本项目的虚拟环境

`uv` 会在项目里创建 `.venv`。它是隔离的 Python 环境，不应该提交到 Git。

`.gitignore` 里应包含：

```gitignore
.venv/
__pycache__/
.pytest_cache/
.ruff_cache/
.future_ledger/
reports/
```

## 常见问题

### `uv run python --version` 不是 3.11

检查是否有 `.python-version`：

```bash
cat .python-version
```

如果没有，创建或更新为：

```text
3.11
```

然后执行：

```bash
uv python install 3.11
uv sync
```

### 安装 AKShare 或 pandas 很慢

这是正常现象，通常是网络或 PyPI 镜像问题。可以先重试：

```bash
uv sync
```

如果公司网络、代理或地区网络导致下载困难，需要配置 Python 包镜像或代理。不要把个人代理配置写进项目文件。

### 真实扫描很慢

v0 计划默认顺序抓取数据，并且优先保证正确性。开发时使用隐藏/开发参数缩小范围：

```bash
uv run future-ledger dividends scan --limit 5
```

### Excel 文件打不开或内容异常

先用测试验证 workbook 结构：

```bash
uv run pytest tests/test_excel_report.py
```

再人工打开 `reports/dividend_rank.xlsx` 检查格式。

## 开发前检查清单

开始编码前，确认以下命令都能正常运行：

```bash
git --version
uv --version
uv python install 3.11
uv run python --version
```

项目骨架完成后，确认：

```bash
uv sync --locked
uv run ruff check .
uv run ruff format --check .
uv run pytest
uv run future-ledger --help
```

如果这些都通过，就具备按照实施计划继续开发 FutureLedger v0 的本地环境了。
